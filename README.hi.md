## मिशन

हमारा मानना है कि पैसा - चाहे वह कहीं भी हो, और चाहे वह किसी भी रूप में गतिमान हो - उन प्रणालियों के समान ही कठोरता का हकदार है जिन्होंने इसे बनाया है। स्मार्ट कॉन्ट्रैक्ट निष्पादित होते हैं। ब्लॉकचेन रिकॉर्ड करते हैं। लेकिन कोई भी *प्रमाणित* नहीं करता।

अटेस्टिया एक अतिरिक्त परत है: संरचनात्मक शासन, नियतात्मक लेखांकन, और मानव-अनुमोदित इरादा - जो सभी श्रृंखलाओं, संगठनों और व्यक्तियों में एकीकृत है।

हम आपके पैसे को नहीं हिलाते हैं। हम यह साबित करते हैं कि क्या हुआ, यह सीमित करते हैं कि क्या हो सकता है, और वित्तीय रिकॉर्ड को अपरिवर्तनीय बनाते हैं।

### हम किस लिए खड़े हैं

- **सत्य, गति से ऊपर।** प्रत्येक वित्तीय घटना अपरिवर्तनीय है, जिसे दोहराया और समायोजित किया जा सकता है। यदि इसे साबित नहीं किया जा सकता है, तो यह नहीं हुआ।
- **मनुष्य अनुमोदन करते हैं; मशीनें सत्यापित करती हैं।** एआई सलाह देता है, स्मार्ट कॉन्ट्रैक्ट निष्पादित करते हैं, लेकिन कुछ भी स्पष्ट मानव प्राधिकरण के बिना नहीं होता। कभी नहीं।
- **संरचनात्मक शासन, राजनीतिक शासन नहीं।** हम यह नहीं तय करते कि क्या मान्य है। हम उन अपरिवर्तनीय नियमों को परिभाषित करते हैं जो बिना शर्त लागू होते हैं - पहचान स्पष्ट है, विरासत अटूट है, क्रम नियतात्मक है।
- **इरादा निष्पादन नहीं है।** आप जो चाहते हैं उसे घोषित करना और उसे करना, ये दो अलग-अलग कार्य हैं जिनके अलग-अलग चरण हैं। उनके बीच की खाई में ही विश्वास मौजूद है।
- **श्रृंखलाएं गवाह हैं, प्राधिकरण नहीं।** XRPL प्रमाणित करता है। एथेरियम निपटान करता है। लेकिन प्राधिकरण संरचनात्मक नियमों से आता है, किसी भी श्रृंखला के सर्वसम्मति से नहीं।
- **सरल बुनियादी ढांचा ही सफल होता है।** दुनिया को एक और DeFi प्रोटोकॉल की आवश्यकता नहीं है। इसकी आवश्यकता उस बुनियादी ढांचे की है जो इसके नीचे है - वह वित्तीय आधारभूत संरचना जो बाकी सब कुछ को विश्वसनीय बनाती है।

## मुख्य पैटर्न

प्रत्येक इंटरैक्शन एक ही प्रक्रिया का पालन करता है:

```
Intent → Approve → Execute → Verify
```

1. **इरादा** - एक उपयोगकर्ता या सिस्टम वांछित परिणाम घोषित करता है
2. **अनुमोदन** - रजिस्ट्रम संरचनात्मक रूप से मान्य करता है; एक मानव स्पष्ट रूप से हस्ताक्षर करता है
3. **निष्पादन** - ऑन-चेन लेनदेन सबमिट किया जाता है
4. **सत्यापन** - समायोजन की पुष्टि करता है; XRPL रिकॉर्ड को प्रमाणित करता है

कोई भी चरण वैकल्पिक नहीं है। कोई भी चरण स्वचालित नहीं किया गया है।

### XRPL एकीकरण परीक्षण

एक स्वतंत्र `rippled` नोड डॉकर में चलता है, जो नियतात्मक ऑन-चेन एकीकरण परीक्षणों के लिए है — किसी भी टेस्टनेट पर निर्भरता नहीं, कोई "फaucet" नहीं, उप-सेकंड में लेजर बंद होना।

```bash
docker compose up -d              # Start standalone rippled
pnpm --filter @attestia/witness run test:integration  # Run on-chain round-trip tests
docker compose down               # Stop rippled
```

## सुरक्षा और डेटा दायरा

- **पहुंच की गई डेटा:** वित्तीय लेजर प्रविष्टियों, प्रमाणीकरण रिकॉर्ड और क्रिप्टोग्राफिक प्रमाणों को पढ़ना और लिखना। जब विटनेस मॉड्यूल सक्रिय होता है, तो यह ब्लॉकचेन नोड्स (XRPL) से जुड़ता है।
- **पहुंच नहीं की गई डेटा:** कोई टेलीमेट्री नहीं। कोई उपयोगकर्ता क्रेडेंशियल भंडारण नहीं। कोई तृतीय-पक्ष विश्लेषण नहीं।
- **आवश्यक अनुमतियाँ:** स्थानीय डेटा निर्देशिकाओं तक पढ़ने/लिखने की पहुंच। केवल ब्लॉकचेन प्रमाणीकरण के लिए नेटवर्क एक्सेस। पूर्ण STRIDE विश्लेषण के लिए [THREAT_MODEL.md](THREAT_MODEL.md) देखें।
